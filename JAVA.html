multiple threads

 Main method is the main thread, and can creat other sub threads. Only if all threads are terminated, the entire process finishes. When running, each thread runs for several time slices, and all threads are executed by some scheduling algorithm.

  thread states

   ready

    already established but not launched yet

   executable

    already launched and only wait for CPU

   inexecutable

    cannot run even if getting CPU

   terminated

    finished normally

  method

   class MyThread inherit class Thread(implements Runnable)

    MyThread Must override run() Thread t = new MyThread(); t.start();

   class Mythread implements Runnable(recommended)

    MyThread Must override run() Thread t = new Thread(new MyThread ()); t.start();

  mutex

   use synchronized to modify a method using critical resource. Then the method in different must run in mutual exclusion

   use wait、notify、notifyall

    They are all member methods of class object.

     wait()

       It can make current thread into inexecutable states. It should be invoked from a resource object shared among threads

     notify()

      Wake up another inexecutable thread which is caused by wait() method in this object.

     notifyall()

      Wake up all inexecutable threads which is caused by wait() method in this object.

     yield()

      Abandon CPU just once

     sleep()

      Enter inexecutable states for a certain period of time
